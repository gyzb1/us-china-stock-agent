## 🔍 为什么看不到新进入公司？

### 原因1: 首次运行

**情况**：
- 这是第一次运行程序
- 系统正在建立历史记录

**解决**：
- 首次运行会记录当前所有Top 30公司
- 明天或下次运行时，如果有新公司进入，就会显示

**说明**：
```
第1次运行（今天）:
- 记录：AAPL, MSFT, NVDA, ... (30家)
- 显示：💤 暂无新进入公司

第2次运行（明天）:
- 当前Top 30: AAPL, MSFT, NEWCO, ...
- 检测到：NEWCO是新公司！
- 显示：🌟 新星公司 - NEWCO
```

---

### 原因2: 确实没有新公司

**情况**：
- 今天的Top 30与昨天完全相同
- 没有新公司进入

**显示**：
```
💤 新星公司追踪
今日暂无新进入Top 30的公司...
```

**说明**：
- 这是正常情况
- 大多数时候Top 30比较稳定
- 只有在市场波动大时才会有新公司进入

---

### 原因3: 历史记录文件被删除

**情况**：
- `data/top30-history.json`文件被删除
- 系统重新开始记录

**解决**：
- 等待下次运行
- 系统会重新建立历史记录

## 📊 查看历史记录

### 历史记录文件位置

```
us-china-stock-agent/
└── data/
    └── top30-history.json  ← 历史记录文件
```

### 查看文件内容

```json
{
  "lastUpdate": "2025-10-30T15:22:00.000Z",
  "symbols": [
    "AAPL", "MSFT", "NVDA", "TSLA", "GOOG",
    "AMZN", "META", "TSM", "AVGO", "ORCL",
    ...
  ],
  "records": [
    {
      "symbol": "NEWCO",
      "firstSeenDate": "2025-10-30",
      "firstSeenRank": 25,
      "firstSeenVolume": 5000000000
    }
  ]
}
```

**字段说明**：
- `lastUpdate`: 最后更新时间
- `symbols`: 所有曾经出现过的公司代码
- `records`: 每个公司首次出现的详细记录

## 🎯 如何触发新进入公司？

### 方法1: 等待市场变化

**自然情况**：
- 某公司发布重大消息
- 成交额突然放大
- 进入Top 30

**示例**：
```
某公司发布财报超预期
→ 股价大涨
→ 成交额暴增
→ 进入Top 30
→ 系统检测到
→ 显示在"新星公司"板块
```

---

### 方法2: 测试功能（开发用）

**清除历史记录**：
```bash
# 删除历史记录文件
rm data/top30-history.json

# 重新运行
npm run dev

# 首次运行：记录当前Top 30
# 第二次运行：如果Top 30有变化，会检测到
```

**注意**：
- 仅用于测试
- 生产环境不要删除历史记录
